## 문제 상황
- candles 데이터가 refetch될 때마다 fitContent()를 호출하고 있었습니다.
- fitContent()는 모든 데이터를 화면에 맞추도록 visible range를 재조정하는 메서드이므로, 사용자가 과거 데이터를 탐색하거나 줌 상태를 조정한 경우에도 viewport가 강제로 변경될 수 있습니다.


## 개선 내용
- 최초 로딩 시에만 fitContent()를 호출하도록 변경했습니다.
- 이후 refetch에서는 현재 사용자의 viewport 상태를 유지하도록 처리했습니다.


## 개선 방법
- `useRef`를 활용해 차트의 최초 로딩 여부를 관리했습니다.
- 최초 로딩 시에는 `fitContent()`를 호출하여 전체 캔들이 화면에 보이도록 했습니다.
- 이후 데이터 refetch 시에는 `fitContent()` 호출을 방지하여 사용자가 탐색 중인 차트 viewport가 초기화되지 않도록 개선했습니다.

## useRef를 사용한 이유

- `useRef`는 렌더링 이후에도 값을 유지할 수 있습니다.
- 값 변경 시 불필요한 re-render가 발생하지 않도록 하여 차트 렌더링 성능과 사용자 경험이 좋아집니다.
