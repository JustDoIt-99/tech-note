### 네트워크란?
- 노드(Node)와 링크(Link)가 서로 연결되어 있으며 리소스를 공유하는 집합이다.  
- 노드: 서버, 라우터, 스위치 등 네트워크 장치
- 링크: 유선 또는 무선

- 좋은 네트워크 → 많은 처리량, 짧은 지연 시간, 적은 장애 빈도

### 처리량이란?
- 링크 내에 성공적으로 전달된 데이터의 양, 얼만큼의 트래픽을 처리했는가? `많은 트래픽을 처리 = 많은 처리량`

- 단위로는 bps(byte per second) 사용, 초당 전송 또는 수신되는 비트 수
- 처리량 영향 요소 : 트래픽, 네트워크 장치 간 대역폭, 네트워크 중간에 발생 에러, 하드웨어 스펙
- 트래픽 = 특정 시점에 링크에 흐르는 데이터의 양
- `트래픽이 많아졌다 = 흐르는 데이터가 많아졌다.`, `처리량이 많아졌다 = 처리되는 트래픽이 많아졌다.`
- 대역폭 : 주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 **최대 비트 수**


### 지연시간이란?
- 요청이 처리되는 시간, 어떤 메시지가 두 장치 사이를 왕복한 시간
- 지연 시간 영향 요소 : 매체 타입(유선, 무선), 패킷 크기, 라우터의 패킷 처리 시간
