### 왜 DFS를 사용했나요?

3개의 숫자를 중복 없이 선택해야 하므로 조합 문제라고 판단을 했고, DFS를 통해 각 단계(depth)에서 하나씩 선택하면서 모든 조합을 탐색했습니다.  


### 시간 복잡도
조합 : O(C(N, 3))
소수 판별 O(√M)

O(C(N, 3) x O(√M))

### 왜 i+1을 넘겨주나요?

이미 선택한 원소 이전의 원소를 다시 선택하지 않기 위해서 입니다. 
이를 통해 중복 없는 조합을 생성할 수 있습니다.  


### selected 배열을 전역으로 둔 이유는?

DFS 호출마다 배열을 새로 생성하지 않고 재사용하여 불필요한 객체 생성을 줄였습니다.  


### 왜 i <= num까지 안돌고 i*i <= num 까지만 검사하나요?

어떤 합성수 N은 √N 보다 작은 약수를 반드시 하나 이상 가지므로,  √N까지만 확인하면 충분하다고 판단했습니다.  


### 보통 백트래킹은 원상복구하는데 왜 없나요?

`selected[depth] = nums[i];` 이후 다음 DFS 호출에서 `selected[depth]`는 항상 덮어써지기 때문에 별도의 복원이 필요 없습니다.  


### `if(num < 2) return true;` 이렇게 했음에도 성공한 이유?

nums의 각 원소는 최소 1보다 작기 때문에 아무리 작은 수라도 3이 됩니다.
